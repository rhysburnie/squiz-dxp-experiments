## Plan: Scaffold Astro adapter and ntg-components packages with hello-world

TL;DR: Scaffold two workspace packages under `packages/`: `squiz-astro-adapter` for the custom Astro adapter, and `ntg-components` as an Astro package that uses it. Add a single `.astro` component `hello-world` with a `who` prop defaulting to `world`.

Further spac 001 describes adopter design

**Steps**
1. Create `packages/squiz-astro-adapter/` with a package manifest and source entry for the custom Astro adapter/integration. Keep dependencies minimal, targeting Astro plugin/hook development.
2. Create `packages/ntg-components/` as an Astro package with `astro` as the only runtime/framework dependency. Add `@squiz-astro-adapter` as a workspace dependency.
3. In `packages/ntg-components/src/`, create `hello-world/hello-world.astro` that uses `Astro.props.who` and defaults it to `world`, rendering `hello ${who}`.
4. Add a basic `packages/ntg-components/astro.config.mjs` and a minimal `package.json` scripts section for `dev` and `build`.
5. Do not add React, Vue, Svelte, or any other client framework dependencies at this stage.

**Relevant files**
- `/workspaces/squiz-dxp-experiments/packages/squiz-astro-adapter/package.json`
- `/workspaces/squiz-dxp-experiments/packages/squiz-astro-adapter/src/index.ts`
- `/workspaces/squiz-dxp-experiments/packages/ntg-components/package.json`
- `/workspaces/squiz-dxp-experiments/packages/ntg-components/astro.config.mjs`
- `/workspaces/squiz-dxp-experiments/packages/ntg-components/src/hello-world/hello-world.astro`
- `/workspaces/squiz-dxp-experiments/packages/ntg-components/src/pages/index.astro` (optional demo or entry)
- `/workspaces/squiz-dxp-experiments/pnpm-workspace.yaml`

**Verification**
1. Confirm `packages/` contains `squiz-astro-adapter` and `ntg-components` directories.
2. Confirm `ntg-components/package.json` depends on the local adapter package and only Astro-related deps.
3. Confirm `packages/ntg-components/src/hello-world/hello-world.astro` renders `hello ${who}` with a `who` default of `world`.
4. Confirm there are no React-related dependencies in root or package manifests.
5. Confirm `ntg-components` can run `pnpm --filter ntg-components dev` or `pnpm --filter ntg-components build` after scaffolding.

**Decisions**
- Use one adapter package and one Astro component package only.
- Keep the adapter package separate and local for future development, but no shared component package yet.
- Keep `ntg-components` Astro-only and focus on the single `hello-world` component.

**Further Considerations**
1. Later, if we want shared reusable components, add a third `packages/components` package.
2. The adapter package should expose a simple integration entrypoint so the Astro package can import it cleanly.
3. Empty root workspace config is fine; no change needed to `turbo.json` or `pnpm-workspace.yaml` unless package filtering is required later.
